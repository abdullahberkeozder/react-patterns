# Contributing Guidelines

Thanks for contributing to this React learning monorepo.

## How to Contribute

1. Fork the repository and create a focused branch.
2. Keep changes scoped to a single feature, pattern, or documentation topic.
3. Update documentation when introducing new folders, apps, or packages.
4. Open a pull request with a clear summary of what changed and why.

## Contribution Standards

- Prefer clear, educational examples over overly clever implementations.
- Keep naming descriptive and consistent.
- Avoid unrelated refactors in the same pull request.
- Add or update tests when the repository test setup exists for the area being changed.

## Pull Request Checklist

- [ ] Change is focused and minimal
- [ ] Relevant documentation is updated
- [ ] Local checks (if configured) pass
- [ ] No unrelated files are modified
